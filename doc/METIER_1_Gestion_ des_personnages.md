# Règles métiers générales concernant les personnages
---------------------------------------------------------
## création d'un personnage
* Un personnage possède obligatoirement 
   - une race et au moins une classe
   - un alignement (choisi parmi les neufs possibles)

## Définition d'une classe de personnage
* une classe est découpées en niveaux (selon le ruleset de 3 à 20). Une clas


# Règles métiers spécifiques au ruleset DD3.5
-------------------------------------------
## Races
* Le ruleset distingue deux types de races : les races de base et les archétypes
* le personnage possède obligatoirement une race de base et possiblement un ou plusieurs archétypes

## Classes
* le ruleset distingue deux types de classes : les classes de base et les classes de prestige
* la première classe du personnage est forcément une classe de base. Par la suite, le personnage peut ajouter d'autres classes de base et/ou de prestige.


# Règles métiers spécifiques au ruleset DD02024
----------------------------------------------
## Races
* Les races s'appelent des espèces dans ce ruleset. Par convention durant le développement de l'appliciation, nous continuerons à parler de race. Il n'y pas pas de distingo entre les races qui sont toutes considérées comme des races de base
* Un personnage ne peut avoir qu'une et une seule race. 

## historique
* le ruleset introduit le concept d'historique qui complète la race et la ou les classes
* un perosnnage possède un et un seul historique, choisi à la création du personnage
